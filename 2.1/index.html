Metadaten DCAT-AP AT

Erläuterungen zur Anpassung von DCAT-AP für Österreich – Version 2.1

Dieses Dokument definiert DCAT-AP AT 2.1, das österreichische Profil von [[DCAT-AP]] zur Beschreibung offener Verwaltungsdaten auf data.gv.at. Es legt fest, welche Metadatenfelder verpflichtend, empfohlen oder optional sind, und richtet sich an Betreiber von Datenportalen sowie alle Stellen, die Metadaten für österreichische Verwaltungsdaten bereitstellen.

Dieses Dokument ist eine Empfehlung (Recommendation) der BLSG Fachgruppe data.gv.at und verbindlich für alle österreichischen Verwaltungsstellen, die Metadaten auf data.gv.at publizieren.

Management Summary

DCAT-AP ist der europäische Metadatenstandard für den Austausch offener Verwaltungsdaten. Er basiert auf der W3C Recommendation [[DCAT]] und definiert ein einheitliches Vokabular zur Beschreibung von Datenkatalogen, Datensätzen und Distributionen.

Österreich hat Open Data früh eingeführt und dabei einen eigenen nationalen Metadatenstandard (metadaten-data.gv.at, Version 2.6) entwickelt. Mit der Verbreitung von DCAT-AP auf europäischer Ebene und der Notwendigkeit interoperabler Metadaten wurde der Auftrag erteilt, den nationalen Standard auf DCAT-AP umzustellen. Das Ergebnis dieser Arbeit ist DCAT-AP AT: ein österreichisches Profil, das DCAT-AP um nationale Anforderungen ergänzt und gleichzeitig die Kompatibilität mit dem europäischen Standard sicherstellt.

Die Standardisierungshierarchie lautet: [[DCAT]] (W3C) → [[DCAT-AP]] (EU) → DCAT-AP AT (Österreich).

Datensatz (dcat:Dataset)

Datendienst (dcat:DataService)

Ein Datendienst (dcat:DataService) ist eine Sammlung von Operationen, die über eine API programmatischen Zugriff auf einen oder mehrere Datensätze ermöglicht – z.B. WFS, WMS, OGC API Features oder SPARQL-Endpoint.

Beschreibung des Endpunkts dcat:endpointDescription empfohlen 0..n URI-Referenz Diese Eigenschaft enthält eine Beschreibung der über die Endpunkte verfügbaren Dienste, einschließlich ihrer Operationen, Parameter usw. Die Eigenschaft enthält spezifische Details zu den tatsächlichen Endpunktinstanzen, während dct:conformsTo dazu verwendet wird, um den allgemeinen Standard oder die allgemeine Spezifikation anzugeben, welche die Endpunkte implementieren.

Format dcterms:format empfohlen 0..n URI-Referenz Diese Eigenschaft bezieht sich auf das Dateiformat des Datendienstes gemäß der EU Vocabularies File Type Named Authority List (https://op.europa.eu/en/web/eu-vocabularies/concept-scheme/-/resource?uri=http://publications.europa.eu/resource/authority/file-type).

Akteur (Herausgeber) (foaf:Agent)

Ein Akteur (foaf:Agent) repräsentiert eine Organisation oder Person, die als Herausgeber oder verantwortliche Stelle eines Datensatzes auftritt. Die Klasse wird über dcterms:publisher referenziert.
